   - Validates that config backups have file paths specified

3. ✅ FULL BACKUP INCLUDES FILES METADATA
   - Backup metadata now tracks which files were backed up
   - File count and paths stored in backup.metadata
   - Works for files, config, and full types

4. ✅ RESTORE LOGIC FIXED FOR ALL TYPES
   - Config type: restores files ONLY (not database)
   - Files type: restores files ONLY (not database)
   - Database type: restores database ONLY (not files)
   - Full type: restores BOTH database and files
   
   CRITICAL FIX: Changed from negative to positive logic
   - Before: restoreDb = backup.type !== "files" (WRONG!)
   - After:  restoreDb = backup.type === "database" || type === "full" (CORRECT!)

5. ✅ RESTORE BUTTON NOW TYPE-AWARE
   - Button uses dynamic restore flags based on backup type
   - No longer hardcoded to restore files only

================================================================================
                         HOW TO USE - FULL BACKUP
================================================================================

1. Click "Create Backup" button
2. Select "Full Backup" type
3. Enter optional backup name
4. ⭐ NEW: Directories/Files section now appears
5. Add server paths (e.g., /var/www/uploads, /var/backups)
6. Click "Create Backup"
7. Monitor timeline - will backup both DB and specified files
8. On restore, both database and files are recovered
